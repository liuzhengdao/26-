## 三、硬件调试情况

| 模块 | 引脚 | 测试结果 | 备注 |
|------|------|:--:|------|
| OLED | PB6(SCL)/PB7(SDA) I2C1 | ✅ 正常 | SH1106 兼容芯片，需列偏移 2 |
| MPU6050 | PB6/PB7 I2C1（与OLED共享） | ✅ 正常 | WHO_AM_I=0x68，I2C 400kHz |
| HC-05 | PA2(TX)/PA3(RX) USART2 | ✅ 正常 | 波特率 9600（HC-05 默认 38400，已适配） |
| EC11 编码器 | PA6(TRA)/PA7(TRB)/PB10(PSH) | ✅ 正常 | TIM3 编码器模式，PB10 EXTI 按键 |
| 电源 | 锂电池→AMS1117→3.3V | ✅ 正常 | Type-C 充电 + 电池供电均正常 |
| ST-Link | SWDIO/SWCLK/GND | ✅ 正常 | 部分板子需降频至 100KHz |

### 遇到的关键硬件问题与解决

| 问题 | 原因 | 解决 |
|------|------|------|
| OLED 黑屏 | I2C 地址未检测 + SH1106 列偏移 | 自动扫描地址 + 偏移 2 |
| OLED 上字符乱码 | 16号字体格式（列优先 vs 行优先） | 统一为列优先格式，修复渲染逻辑 |
| MPU6050 + OLED 共用 I2C1 黑屏 | 面包板并线接触不良 | 独立接线，确保 GND 共地 |
| 蓝牙数据乱码 | HC-05 波特率与 STM32 不匹配 | 统一为模块实际波特率 |
| FreeRTOS 加入后屏幕黑屏、启动卡死 | `osDelay()` 在调度器启动前被调用，导致硬件错误 | 初始化阶段改用 `HAL_Delay()`，任务内部保留 `osDelay()` |
| 调度器启动成功但所有任务无法延时 | FreeRTOS SysTick 控制寄存器未使能（CTRL=0x0010），滴答中断不触发 | 在 `MX_FREERTOS_Init` 中手动配置 SysTick 寄存器 |
| MPU6050 初始化时系统卡死 | I2C 通信超时参数为 `HAL_MAX_DELAY`（无限等待），设备未应答时永久阻塞 | 超时参数改为 200ms，超时后返回错误码继续运行 |
| 多任务并发时 OLED 渲染卡死 | OLED 与传感器任务各对同一互斥锁加锁两次，造成死锁 | 去除外层冗余加锁，仅在内层 I2C 操作函数中保护 |
| 任务栈与堆空间不足导致创建失败 | 5 个任务栈合计约 12KB，超出默认堆 8KB | 增大 `configTOTAL_HEAP_SIZE` 至 10240，同时精简各任务栈大小 |

### 重大 Bug 排查记录

| # | 问题 | 根因 | 解决 |
|---|------|------|------|
| 1 | FreeRTOS 添加后屏幕黑屏 | `osDelay()` 在 `osKernelStart()` 前调用 | 改为 `HAL_Delay()` |
| 2 | 屏幕卡在 "System Init..." | `MPU6050_Init()` 中 `HAL_MAX_DELAY` 无限等待 | 改为 200ms 超时 |
| 3 | 调度器启动后任务不运行 | FreeRTOS SysTick 未使能（CTRL=0x10） | `MX_FREERTOS_Init` 中手动配置 SysTick |
| 4 | OLED 渲染线程死锁 | 任务各锁两次互斥锁 | 去掉外层锁，内层已保护 |
| 5 | 浮点数值显示为空 | newlib-nano 不支持 `%f` | 实现 `fmt_fixed()` 整数格式化 |
| 6 | 堆溢出导致任务创建失败 | `configTOTAL_HEAP_SIZE` 不足 | 增至 10240 + 缩任务栈 |
| 7 | 蓝牙输出全 0x00 | HC-05 波特率不匹配 | 匹配模块实际波特率 |
